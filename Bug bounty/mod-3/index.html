Módulo 3 - Introducción al Linux

Linux es un sistema operativo, es decir, un programa que se encarga de manejar los recursos de una computadora y administrar las aplicaciones que se ejecutan en ella. Es similar a otros sistemas operativos como Windows o macOS, pero con algunas diferencias importantes.

Linux es un sistema operativo de código abierto, lo que significa que su código fuente está disponible para cualquier persona que quiera usarlo o modificarlo. Fue creado por Linus Torvalds en 1991 y desde entonces ha sido utilizado por millones de personas en todo el mundo.

Una de las principales ventajas de Linux es su estabilidad y seguridad. Ya que es de código abierto, los usuarios pueden revisar y corregir cualquier error o vulnerabilidad que encuentren. Además, Linux es altamente personalizable y puede ser adaptado para satisfacer las necesidades específicas de cada usuario.

Otra gran ventaja de Linux es su amplia variedad de distribuciones. Cada distribución de Linux tiene su propio conjunto de características y herramientas, lo que permite a los usuarios elegir la que mejor se adapte a sus necesidades.

En Linux, el núcleo o “kernel” es el corazón del sistema operativo y es responsable de controlar los recursos de la computadora.

Estructura del Linux

Linux usa una estructura de árbol y se organiza en una jerarquía de directorios, donde cada directorio tiene un propósito específico. El directorio raíz, representado por el símbolo "/", es el nivel superior de la jerarquía.

Algunos de los directorios más comunes en la jerarquía de Linux incluyen:

/devContiene los archivos de dispositivo para los dispositivos conectados al sistema.

Linux Shell

El "Shell" es una interfaz de línea de comandos que permite a los usuarios interactuar con el sistema operativo. Es una herramienta poderosa que permite a los usuarios realizar diversas tareas, como crear y eliminar archivos, navegar por el sistema de archivos y cambiar la configuración del sistema. En lugar de hacer clic en los iconos y menús de una interfaz gráfica de usuario, el shell permite a los usuarios escribir comandos directamente en una terminal.

Por ejemplo, para crear un archivo en Linux, se puede utilizar el comando "touch" seguido del nombre del archivo:

$ touch archivo.txt

El shell es altamente personalizable y permite a los usuarios crear scripts y automatizar tareas. Por ejemplo, se puede crear un script que realice automáticamente una copia de seguridad de los archivos importantes del sistema.

El shell también proporciona una gran cantidad de herramientas y utilidades, como "grep" para buscar patrones en archivos, "sed" para editar texto y "awk" para procesar datos.

Si quieres saber que shell estás usando, puedes usar estos comandos:

$ echo "$SHELL"

o puedes usar:

$ ps -p $$

Permisos de archivos en Linux

En Linux, cada archivo y directorio tiene tres tipos de permisos: lectura, escritura y ejecución:

Estos permisos pueden ser otorgados a tres tipos de usuarios: el propietario del archivo, los miembros del grupo al que pertenece el archivo y otros usuarios.

La estructura es asi:

Para ver los permisos de un archivo o directorio, se utiliza el comando "ls -l". El resultado incluirá una serie de caracteres que indican los permisos para cada tipo de usuario. Por ejemplo, la siguiente salida indica que el archivo "myfile" es propiedad del usuario "user1" y tiene permisos de lectura y escritura para el usuario, pero solo permisos de lectura para los miembros del grupo y otros usuarios:

-rw-r--r-- 1 user1 user1 1234 Aug 31 14:23 myfile

Los permisos se indican con una serie de caracteres: "r" para lectura, "w" para escritura y "x" para ejecución. Además, se utilizan otros caracteres especiales para indicar si el archivo es un directorio, un archivo especial o un enlace simbólico.

Para cambiar los permisos de un archivo o directorio, se utiliza el comando "chmod". Por ejemplo, para dar permisos de escritura y ejecución a todos los usuarios en el archivo "myfile", se puede utilizar el siguiente comando:

chmod a+wx myfile

En este caso, "a" indica que se están cambiando los permisos para todos los usuarios, "+" indica que se están agregando permisos y "wx" indica que se están agregando permisos de escritura y ejecución.

Los permisos también se pueden especificar utilizando números en lugar de caracteres. Cada permiso tiene un valor numérico asociado: 4 para lectura, 2 para escritura y 1 para ejecución.

Tabla de valores decimales de Permisos de Linux

NúmeroPermisoLetras
0Ningún permiso---
1Ejecución--x
2Escritura-w-
3Escritura y ejecución-wx
4Lecturar--
5Lectura y ejecuciónr-x
6Lectura y escriturarw-
7Lectura, escritura y ejecuciónrwx

Por ejemplo, para dar permisos de lectura y escritura al usuario y permisos de ejecución a todos los usuarios en el archivo "myfile", se puede utilizar el siguiente comando:

chmod 754 myfile

En este caso, el primer dígito "7" indica los permisos para el propietario del archivo (lectura, escritura y ejecución), el segundo dígito "5" indica los permisos para los miembros del grupo (lectura y ejecución) y el tercer dígito "4" indica los permisos para otros usuarios (lectura).

Cambiar el Propietario de un Archivo

Para cambiar el propietario de un archivo en Linux, se utiliza el comando "chown" seguido del nombre del nuevo propietario y el nombre del archivo. Por ejemplo, si queremos cambiar el propietario del archivo "myfile" a "user2", podemos utilizar el siguiente comando:

sudo chown user2 myfile

En este caso, "sudo" es necesario para obtener los permisos de superusuario necesarios para cambiar el propietario del archivo; "user2" es el nombre del nuevo propietario y "myfile" es el nombre del archivo que deseamos cambiar.

También es posible cambiar el grupo al que pertenece el archivo utilizando el mismo comando. Para hacer esto, se utiliza la opción "-R" seguida del nombre del nuevo propietario y el nombre del archivo. Por ejemplo, si queremos cambiar el propietario del archivo "myfile" y todos los archivos y directorios dentro de él a "user2" y al grupo "group2", podemos utilizar el siguiente comando:

sudo chown -R user2:group2 myfile

Es importante tener en cuenta que solo los usuarios con permisos de superusuario pueden cambiar el propietario de un archivo o directorio. Además, cambiar el propietario de un archivo puede afectar la accesibilidad y la funcionalidad del archivo, por lo que se recomienda tener cuidado al realizar estos cambios.
