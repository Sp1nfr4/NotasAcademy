Pruebas de Configuración y Gestión de despliegue

Por medio de estas pruebas podemos lograr la identificación de problemas como la exposición de información confidencial en los archivos de configuración, la falta de control de versiones adecuado y la posibilidad de que se introduzcan cambios maliciosos en el sistema durante el proceso de despliegue.

Un ejemplo de vulnerabilidad en la gestión de la configuración podría ser si un archivo de configuración de una aplicación web incluyera contraseñas o claves de acceso en texto plano. Esto podría permitir a un atacante acceder a información confidencial, como datos de usuarios o credenciales de administrador.

Otro ejemplo de vulnerabilidad en la gestión del despliegue podría ser si se permitiera a un usuario introducir cambios maliciosos en el sistema durante el proceso de despliegue. Esto podría permitir a un atacante introducir vulnerabilidades en el sistema o incluso tomar el control completo del sistema.

Para identificar estas vulnerabilidades, se pueden utilizar diferentes técnicas, como la revisión de archivos de configuración y el análisis de los registros de implementación y despliegue. También se pueden realizar pruebas de inyección de código para ver si se pueden introducir cambios maliciosos en el sistema durante el proceso de despliegue.

3. Pruebas a Extensiones de Archivo

Algunas de las extensiones de archivo que NO deberian estar siendo publicadas en un servidor web, las cuales debemos buscar son:

  • .asa
  • .inc
  • .config

Algunas de las extensiones que debemos buscar por que contienen información que podria sernos útil:

  • Archivos comprimidos: .zip, .tar, .gz, .tgz, .rar, .gzip, .rte, .7z, etc.

Otras Extensiones que podemos buscar son:

  • Archivos de Texto: .txt, .rtf
  • Documentos PDF: .pdf
  • Documentos de Office (OLE - Object Linked and Embedded): .doc, .xls, .ppt
  • Archivos de Copia de Seguridad o sus Indicativos: .bak, .old, .copy
  • Archivos de Código fuente de Java: .java
