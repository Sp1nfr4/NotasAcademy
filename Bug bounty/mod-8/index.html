Módulo 8 - Pruebas de gestión de identidades

La autenticación es el proceso de verificar la identidad de un usuario, mientras que el control de acceso es el proceso de determinar si un usuario tiene permiso para acceder a un recurso.

Es importante asegurarse de que la autenticación se realice de manera segura y que no se puedan eludir los controles de acceso. Algunas vulnerabilidades comunes en este ámbito incluyen:

  • Contraseñas débiles o predecibles
  • Fallos en la implementación de autenticación multifactor
  • Sesiones no finalizadas adecuadamente
  • Sesiones robadas o secuestradas
  • Ataques de fuerza bruta

Para identificar estas vulnerabilidades, se pueden utilizar diferentes técnicas, como probar contraseñas comunes o intentar iniciar sesión con credenciales robadas. También se pueden realizar pruebas de sesión, como intentar acceder a recursos sin iniciar sesión o intentar acceder a recursos después de cerrar sesión.
